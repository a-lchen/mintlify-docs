---
title: Configurações Globais
description: Mintlify oferece controle total sobre a aparência da sua documentação usando o arquivo docs.json
icon: gear
---

Cada site Mintlify requer um`docs.json` arquivo com as configurações principais. Aprenda mais sobre as[propriedades](#properties) abaixo.

## Propriedades

<ResponseField name="name" type="string" required>
  Nome do seu projeto. Usado para o título global.

  Exemplo:`mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Um array de grupos com todas as páginas dentro desse grupo

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      O nome do grupo.

      Exemplo:`Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Os caminhos relativos para os arquivos Markdown que servirão como páginas.

      Exemplo:`["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Caminho para a imagem do logo ou objeto com caminhos para imagens do logo para os modos 'light' e 'dark'

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Caminho para o logo no modo light
    </ResponseField>

    <ResponseField name="dark" type="string">
      Caminho para o logo no modo dark
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      Para onde o clique no logo direciona
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Caminho para a imagem do favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de cores hex para seu tema global

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      A cor primária. Mais comumente usada para conteúdo destacado, cabeçalhos de seção e acentos no modo light
    </ResponseField>

    <ResponseField name="light" type="string">
      A cor primária para o modo dark. Mais comumente usada para conteúdo destacado, cabeçalhos de seção e acentos no modo dark
    </ResponseField>

    <ResponseField name="dark" type="string">
      A cor primária para botões importantes
    </ResponseField>

    <ResponseField name="background" type="object">
      A cor do fundo tanto no modo light quanto no dark

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          O código de cor hex do fundo no modo light
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          O código de cor hex do fundo no modo dark
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de`name`s e`url`s de links que você deseja incluir na barra superior

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      O nome do botão.

      Exemplo:`Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      A URL ao clicar no botão. Exemplo:`https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      Link mostra um botão. GitHub mostra as informações do repositório na URL fornecida, incluindo o número de estrelas no GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Se`link`: Para onde o botão vincula.

      Se`github`: Link para o repositório do qual carregar informações do GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texto dentro do botão. Apenas necessário se`type` é um`link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nomes de versões. Use isso apenas se você quiser mostrar diferentes versões da documentação com um dropdown na barra de navegação.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Uma matriz de âncoras, contém as`icon`,`color`, e`url`.

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      O[Font Awesome](https://fontawesome.com/search?q=heart)ícone usado para a âncora.

      Exemplo:`comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      O nome do rótulo da âncora.

      Exemplo:`Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      O início da URL que indica quais páginas pertencem à âncora. Normalmente, este é o nome da pasta onde você armazena suas páginas.
    </ResponseField>

    <ResponseField name="color" type="string">
      A cor hexadecimal do fundo do ícone da âncora. Também pode ser um gradiente se você passar um objeto com as propriedades`from`e`to`que são cores hexadecimais.
    </ResponseField>

    <ResponseField name="version" type="string">
      Usado quando você deseja ocultar uma âncora até que a versão correta dos documentos seja selecionada.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passe`true`se você quiser ocultar a âncora até que alguém vincule diretamente aos documentos dentro dela.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Um dos seguintes: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Substitua as configurações padrão para a âncora superior.

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      O nome da âncora superior
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Ícone Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Um dos seguintes: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Uma matriz de abas de navegação.

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      O nome do rótulo da aba.
    </ResponseField>

    <ResponseField name="url" type="string">
      O início da URL que indica quais páginas pertencem à aba. Normalmente, este é o nome da pasta onde você armazena suas páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuração para configurações de API. Saiba mais sobre páginas de API em[Componentes de API](/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      A URL base para todos os endpoints da API. Se`baseUrl`for uma matriz, permite múltiplas opções de URL base para o usuário escolher.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          A estratégia de autenticação usada para todos os endpoints da API.
        </ResponseField>

        <ResponseField name="name" type="string">
          O nome do parâmetro de autenticação usado no playground da API.

          Se o método for`basic`, o formato deve ser`[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          O valor padrão destinado a ser o prefixo para o campo de entrada de autenticação.

          Por exemplo, se`inputPrefix`for`AuthKey`o resultado padrão da entrada de autenticação herdaria como`AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurações para o playground da API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Se o playground é mostrado, oculto ou apenas mostra o endpoint sem interatividade adicional do usuário`simple`

          Saiba mais nos[guias do Playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Ativar esta flag garante que a ordem das chaves nas páginas OpenAPI corresponda à ordem das chaves definida no arquivo OpenAPI.

      <Warning>Este comportamento será ativado por padrão em breve, após o qual este campo será marcado como obsoleto.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Uma string ou matriz de strings de URL(s) ou caminho(s) relativo(s) que apontam para seu arquivo OpenAPI.

  Exemplos:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Um objeto de contas de mídia social onde o par chave:propriedade representa a plataforma de mídia social e a URL da conta.

  Exemplo:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      Um dos seguintes valores`website`,`facebook`,`x`,`discord`,`slack`,`github`,`linkedin`,`instagram`,`hacker-news`

      Exemplo:`x`
    </ResponseField>

    <ResponseField name="property" type="string">
      A URL para a plataforma de mídia social.

      Exemplo:`https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurações para ativar botões de feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Ativa um botão que permite aos usuários enviar sugestões de alterações através de pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Ativa um botão que permite aos usuários relatar um problema com a documentação
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personalize o interruptor do modo escuro.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Defina se você deseja sempre mostrar o modo claro ou escuro para novos usuários. Se não definido, usaremos por padrão o mesmo modo que o sistema operacional do usuário.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Defina como true para ocultar o interruptor de modo claro/escuro. Você pode`isHidden`com`default`para forçar sua documentação a usar apenas o modo claro ou escuro. Por exemplo:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Uma imagem de fundo para ser exibida atrás de cada página. Veja exemplo com[Infisical](https://infisical.com/docs)e[FRPC](https://frpc.io).
</ResponseField>
