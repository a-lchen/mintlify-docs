Il menu di navigazione è l'elenco dei link su ogni sito.

Probabilmente lo aggiornerai`docs.json`ogni volta che aggiungi una nuova pagina. Le pagine non appaiono automaticamente.

## Sintassi di navigazione

La nostra sintassi di navigazione è ricorsiva, il che significa che puoi creare gruppi di navigazione annidati. Non devi includere`.mdx`nei nomi delle pagine.

<CodeGroup>
  ```json Regular Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json Nested Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

## Cartelle

Basta posizionare i tuoi file MDX in cartelle e aggiornare i percorsi in`docs.json`.

Per esempio, per avere una pagina sotto`https://yoursite.com/your-folder/your-page`, creeresti una cartella chiamata`your-folder`contenente un file MDX chiamato`your-page.mdx`.

<Warning>
  Non puoi usare`api`come nome per una cartella a meno che non sia annidata in un'altra cartella. Mintlify usa Next.js, che usa il`api`Cartella riservata per le chiamate server interne. Un nome di cartella come`api-reference`sarebbe accettato.
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Pagine nascoste

I file MDX che non sono inclusi in`docs.json`non vengono visualizzati nella barra laterale, ma sono accessibili tramite la barra di ricerca e il collegamento diretto.
