Il menu di navigazione è l'elenco dei link su ogni sito web.

Probabilmente aggiornerai `docs.json` ogni volta che aggiungi una nuova pagina. Le pagine non appaiono automaticamente.

## Sintassi di navigazione

La nostra sintassi di navigazione è ricorsiva, il che significa che puoi creare gruppi di navigazione annidati. Non è necessario includere `.mdx` nei nomi delle pagine.

<CodeGroup>
  ```json Regular Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json Nested Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

## Cartelle

Basta mettere i tuoi file MDX nelle cartelle e aggiornare i percorsi in `docs.json`.

Per esempio, per avere una pagina in `https://yoursite.com/your-folder/your-page` dovresti creare una cartella chiamata `your-folder` contenente un file MDX chiamato `your-page.mdx`.

<Warning>
  Non puoi usare `api` come nome di una cartella a meno che non la annidi all'interno di un'altra cartella. Mintlify usa Next.js che riserva la cartella `api` di primo livello per le chiamate server interne. Un nome di cartella come `api-reference` sarebbe accettato.
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Pagine nascoste

I file MDX non inclusi in `docs.json` non appariranno nella barra laterale ma sono accessibili attraverso la barra di ricerca e collegandosi direttamente ad essi.
