<Info>
  **前提条件**请在继续之前安装 Node.js（版本 19 或更高）。<br />

  请更新到`docs.json`在继续之前，请删除过时的内容`mint.json`文件。
</Info>

请按照以下步骤在您的操作系统上安装和运行 Mintlify：

**步骤 1**：安装 Mintlify：

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**步骤 2**: 导航到文档目录（其中`docs.json`文件所在的目录下，然后执行以下命令：

```bash
mintlify dev
```

您的文档的本地预览将可在此处获得`http://localhost:3000`。

### 自定义端口

默认情况下，Mintlify 使用 3000 端口。您可以使用`--port`调整标志。例如，要在端口 3333 上运行 Mintlify，请使用以下命令：

```bash
mintlify dev --port 3333
```

如果您尝试在已占用的端口上运行 Mintlify，将会使用下一个可用的端口：

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify版本

请注意，每个 CLI 版本都与特定版本的 Mintlify 相关联。如果您的本地网站与生产版本不匹配，请更新 CLI：

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

## 检查链接

CLI 可帮助检查文档中的引用链接。要识别错误链接，请使用以下命令：

```bash
mintlify broken-links
```

## 故障排除

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    这可能是由于使用过时版本的Node造成的。请尝试以下操作：

    1. 移除当前安装的Mintlify版本：`npm remove -g mintlify`
    2. 更新到 Node v19 或更高版本。
    3. 重新安装 Mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    解决方案：进入您的设备的根目录，删除 \~/.mintlify 文件夹。然后执行`mintlify dev`请再次运行。
  </Accordion>
</AccordionGroup>

好奇 Mintlify CLI 版本发生了什么变化？[查看 CLI 更改日志。](https://www.npmjs.com/package/mintlify?activeTab=versions)
