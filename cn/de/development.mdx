<Info>
  **前提条件**：请在继续之前安装Node.js（版本19或更高）。<br />

  请更新到`docs.json`后继续，并删除过时的`mint.json`文件。
</Info>

按照以下步骤在您的操作系统上安装和运行Mintlify：

**步骤1**：安装Mintlify：

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**步骤2**：导航到文档目录（`docs.json`文件所在位置）并运行以下命令：

```bash
mintlify dev
```

您的文档的本地预览将在`http://localhost:3000`上可用。

### 自定义端口

默认情况下，Mintlify使用端口3000。您可以使用`--port`标志自定义Mintlify运行的端口。例如，要在端口3333上运行Mintlify，使用此命令：

```bash
mintlify dev --port 3333
```

如果您尝试在已被占用的端口上运行 Mintlify，将使用下一个可用端口：

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify 版本

请注意，每个 CLI 版本都与特定版本的 Mintlify 相关联。如果您的本地网站与生产版本不匹配，请更新 CLI：

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

## 检查链接

CLI 可以帮助检查您文档中的引用链接。要识别损坏的链接，请使用以下命令：

```bash
mintlify broken-links
```

## 故障排除

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    这可能是由于 Node 版本过时导致的。请尝试以下操作：

    1. 删除当前安装的 Mintlify 版本：`npm remove -g mintlify`
    2. 更新到 Node v19 或更高版本。
    3. 重新安装 Mintlify：`npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    解决方案：转到设备的根目录并删除 \~/.mintlify 文件夹。之后重新运行`mintlify dev`。
  </Accordion>
</AccordionGroup>

想了解 CLI 版本有什么变化？[查看 CLI 更新日志。](https://www.npmjs.com/package/mintlify?activeTab=versions)
