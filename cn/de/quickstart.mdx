## 设置开发环境

了解如何在本地更新和公开部署您的文档。

### 编辑和预览

<AccordionGroup>
  <Accordion icon="github" title="Clone your docs locally">
    在入职过程中，我们在你的Github上创建了一个包含你文档内容的存储库。你可以在我们的平台上查看这个存储库。[仪表板](https://dashboard.mintlify.com)。要在本地克隆该存储库，请按照以下步骤[指示](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)在您的终端中。
  </Accordion>

  <Accordion icon="rectangle-terminal" title="Preview changes">
    预览功能帮助您确保更改如预期般显示。我们开发了一个命令行接口，以便在本地查看这些更改。

    1. 安装[Mintlify CLI](https://www.npmjs.com/package/mintlify)以本地查看文档更改：`npm i -g mintlify`
    2. 在您的文档主目录中执行以下命令（在`docs.json` ist):&#x20;
       `mintlify dev`

    <Note>
      如果您目前仍在使用早期版本`mint.json`请使用配置文件更新Mintlify CLI：

      `npm i -g mintlify@latest`

      并在您的 Docs 仓库中执行新的升级命令：

      `mintlify upgrade`您现在应该已经看到新的`docs.json`使用配置文件。您可以`mint.json`从您的存储库中删除文件。
    </Note>
  </Accordion>
</AccordionGroup>

### 部署更改

<AccordionGroup>
  <Accordion icon="message-bot" title="Install our Github app">
    我们的 Github 应用会自动将您的更改部署到您的文档页面上，因此您无需自行处理部署。在您的[仪表板](https://dashboard.mintlify.com)一旦机器人成功安装，在仓库的提交哈希旁边应该会出现一个勾号。
  </Accordion>

  <Accordion icon="rocket" title="Push your changes">
    [提交并推送您的更改到 Git](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)以便您的更改能够在您的文档页面上更新。如果您推送并未看到
    Github 应用程序成功部署了您的更改，您也可以通过我们手动
    更新您的文档[仪表盘](https://dashboard.mintlify.com)更新。
  </Accordion>
</AccordionGroup>
