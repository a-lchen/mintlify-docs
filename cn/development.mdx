<Info>
  **前提条件**请在继续之前安装 Node.js（版本 19 或更高）。<br />

  请升级到
  `docs.json` 在继续之前并删除旧版`mint.json`文件。
</Info>

按照以下步骤在您的操作系统上安装和运行Mintlify：

**步骤 1**安装 Mintlify：

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**步骤 2**：导航到 docs 目录（在此处`docs.json` 文件所在位置）并执行以下命令：

```bash
mintlify dev
```

本地预览您的文档可在`http://localhost:3000`。

### 自定义端口

默认情况下，Mintlify 使用端口 3000。您可以通过使用`--port` 标志。例如，要在端口3333上运行Mintlify，请使用以下命令：

```bash
mintlify dev --port 3333
```

如果你尝试在已经被占用的端口上运行 Mintlify，它将使用下一个可用端口：

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify 版本

请注意，每个 CLI 版本都与特定版本的 Mintlify 相关联。 如果您的本地网站与生产版本不一致，请更新 CLI：

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

## 验证链接

CLI可以帮助验证文档中创建的引用链接。若要识别任何损坏的链接，请使用以下命令：

```bash
mintlify broken-links
```

## 代码格式化

我们建议在您的IDE上使用扩展程序以识别和格式化MDX。如果您是VSCode用户，请考虑[MDX VSCode 扩展](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)用于语法高亮，和[Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)用于代码格式化。

## 故障排除

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    这可能是由于 node 版本过旧导致的。请尝试以下操作：

    1. 卸载当前安装的 mintlify 版本：`npm remove -g mintlify`
    2. 升级到 Node v19 或更高版本。
    3. 重新安装mintlify：`npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    解决方案：前往设备的根目录，删除 \~/.mintlify 文件夹。之后，运行`mintlify dev` 再次。
  </Accordion>
</AccordionGroup>

想知道 CLI 版本有什么变化吗？[查看 CLI 更新日志。](https://www.npmjs.com/package/mintlify?activeTab=versions)
