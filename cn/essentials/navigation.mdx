title: 导航
description: docs.json中的导航字段定义了导航菜单中的页面

导航菜单是每个网站上的链接列表。

您可能需要更新`docs.json`每次添加新页面时。页面不会自动显示。

## 导航语法

我们的导航语法是递归的，这意味着您可以创建嵌套的导航组。您不需要包含`.mdx`在页面名称中。

<CodeGroup>
  ```json Regular Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json Nested Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

## 文件夹

只需将您的MDX文件放在文件夹中并更新`docs.json`中的路径。

例如，要在`https://yoursite.com/your-folder/your-page`创建页面，您需要创建一个名为`your-folder` 包含一个MDX文件叫做 `your-page.mdx`。

<Warning>
  你不能使用 `api` 作为文件夹名称，除非将其嵌套在另一个文件夹中。Mintlify 使用 Next.js，它保留顶级 `api` 文件夹用于内部服务器调用。像 `api-reference` 这样的文件夹名称是可以接受的。
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## 隐藏页面

未包含在 `docs.json` 中的MDX文件不会显示在侧边栏中，但可以通过搜索栏和直接链接访问。
